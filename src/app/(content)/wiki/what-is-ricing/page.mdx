# What is Ricing?

"Ricing" refers to the practice of customizing the look and feel of your Linux (or Unix-like) desktop environment. The term originates from car culture — "ricing out" a car with aesthetic modifications — and has been adopted by the Linux community to describe desktop customization.

## Why Rice?

- **Aesthetics** — Create a desktop that looks exactly how you want it
- **Productivity** — Tailor your workflow with tiling window managers, custom keybindings, and status bars
- **Learning** — Gain a deeper understanding of how Linux desktop environments work under the hood
- **Community** — Share your setups on r/unixporn and get inspired by others

## Common Components of a Rice

A typical rice involves customizing some or all of these:

| Component      | Examples                               |
| -------------- | -------------------------------------- |
| Window Manager | i3, Hyprland, bspwm, sway              |
| Terminal       | Alacritty, Kitty, WezTerm              |
| Shell          | zsh, fish, bash                        |
| Bar / Panel    | Waybar, Polybar, eww                   |
| Launcher       | Rofi, wofi, dmenu                      |
| Notifications  | dunst, mako, swaync                    |
| Color Scheme   | Catppuccin, Gruvbox, Nord, Tokyo Night |
| Fonts          | JetBrains Mono, Iosevka, Fira Code     |

## Getting Started

The best way to start is to pick a [Window Manager](/wiki/window-managers) and learn how to manage your configuration with [Dotfiles](/wiki/dotfiles).
